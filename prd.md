# Product Requirements Document (PRD): StyleMark
## 1. Project Overview
**Name:** StyleMark
**Concept:** A purely client-side visual design system configurator. It bridges the gap between non-technical clients and developers by providing an interactive UI to define design tokens (colors, typography, geometry) and exporting a highly structured Markdown file.
**Primary Goal:** To eliminate subjective design back-and-forth by generating a definitive "Source of Truth" .md document that serves as a direct prompt for developers and autonomous AI coding agents.
## 2. Technical Architecture & Environment
StyleMark is a zero-backend, fully client-side application.
 * **Framework:** React powered by Vite for rapid HMR and optimal client-side performance.
 * **Environment:** Node.js v22.
 * **Package Manager:** pnpm exclusively.
 * **UI Foundation:** Tailwind CSS combined with Shadcn UI for high-quality, accessible interactive components.
 * **State Management:** Zustand for lightweight, scalable state handling across the split-pane architecture.
 * **Iconography:** Lucide-react.
 * **Hosting:** Deployable as static assets (Vercel, GitHub Pages, etc.).
## 3. Core Layout & User Interface
The application utilizes a split-pane layout inspired by theme editors (e.g., TweakCN).
### 3.1 Left Pane: The Control Center
A scrollable sidebar divided into logical design sprints.
 * **Global Settings:** Project name input and Base Theme selection (Light, Dark, System).
 * **Vibe Presets:** One-click starting points, specifically featuring an "Elegant Luxury" default (deep dark backgrounds, subtle muted borders, high-contrast typography) alongside "Minimalist SaaS" and "Playful".
 * **Color System:** Hex input pickers for Primary, Secondary, Accent, Background, Foreground, and Semantic UI colors (Success, Warning, Destructive). Includes an auto-generator for 50-900 shade scales.
 * **Typography:** Google Fonts integration. Separate selection for Headings and Body. Base rem scaling slider.
 * **Geometry & Borders:** Global border-radius slider mapping to Tailwind's sm, md, lg, xl, full. Border thickness controls.
 * **Effects & Animation:** Shadow depth toggles (Flat, Elevated, Floating). Transition easing dropdowns.
### 3.2 Right Pane: The Live Preview Dashboard
A reactive, functional dashboard mockup that updates instantly based on Left Pane state changes.
 * **Real-time Variable Injection:** The pane wraps its content in a provider that translates Zustand state into inline CSS variables (e.g., --radius, --primary) applied to the local Tailwind configuration.
 * **Responsive Viewport:** Toggles at the top to constrain the dashboard width to Mobile, Tablet, or Desktop.
 * **Dashboard Components:** * Sidebar navigation and top utility bar.
   * Data visualization cards (mock analytics charts).
   * A paginated data table to test row borders and text density.
   * A "Component Kitchen Sink" tab displaying isolated buttons (Default, Ghost, Outline), form inputs, and badges in various states (Hover, Active, Focus, Disabled).
 * **Quick Mode Flip:** A floating action button to instantly test the current token set in inverted Light/Dark modes.
## 4. Functional Requirements
### 4.1 State Persistence
 * **Auto-Save:** The Zustand store must sync with browser localStorage so users do not lose progress on reload.
 * **Session Portability:** Implement "Export State" and "Import State" buttons allowing the user to download a .json file of their current configuration and upload it later.
### 4.2 The Export Engine (Markdown Generation)
When the user clicks "Generate StyleMark", the application compiles the visual state into a downloadable [ProjectName]-StyleMark.md file.
## 5. Output Specification: The Markdown File Structure
The generated markdown must be strictly formatted to act as a direct prompt for an AI agent (such as Claude Code or OpenClaw) or a human developer. It will contain:
**Block 1: Project Meta & Agent Directives**
 * Project Name.
 * Strict instruction set: "Initialize project using React, Vite, Tailwind CSS, Shadcn UI, and pnpm."
**Block 2: Design Tokens (CSS Variables)**
 * A formatted code block containing the exact :root and .dark CSS variables generated by the client's color and geometry selections.
**Block 3: Tailwind Configuration Map**
 * A JSON-formatted code block representing the extend object for tailwind.config.js, mapping the CSS variables to Tailwind utility classes.
**Block 4: Typography Specifications**
 * Links to the required Google Fonts.
 * Explicit instructions on base font sizes, heading hierarchies, and font weights.
**Block 5: Component Guidelines**
 * Hard rules for UI elements based on user selections (e.g., "All primary action buttons must utilize --radius and apply shadow-md on hover").
## 6. Implementation Phases
 * **Phase 1: Foundation & State.** Setup Vite, Tailwind, Shadcn, and Zustand. Build the JSON state structure that will hold all design tokens.
 * **Phase 2: Split-Pane UI.** Build the Left Pane control inputs and the Right Pane static dashboard.
 * **Phase 3: Reactivity Engine.** Map the Zustand state to CSS variables so the Right Pane updates live as inputs change.
 * **Phase 4: Export Logic.** Write the utility that formats the state into the strict Markdown template and triggers the browser download.
